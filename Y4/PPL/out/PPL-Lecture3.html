The untyped λ\lambdaλ-Calculus is the smallest universal programming language first developed by Alonzo Church in 1932.

It has the same expressive power as Turing machines. It lends itself more the the study of properties of computation than Turing machines do.

The λ\lambdaλ-Calculus contains terms (syntax) and rules to compute terms

Syntax

The λ\lambdaλ-Calculus has only 3 constructs:

  1. Variables
  2. Functions
  3. (Function) Applications

Formal definition

let V\mathcal{V}V be a set of variables
let x,y,z,…x,y,z,\ldotsx,y,z,… range over V\mathcal{V}V

The following Backus-Naur form inductively defines the syntax:

M::=x∣λx.M∣MMM ::= x | \lambda x.M | M MM::=x∣λx.M∣MM

**terms can be surrounded with parenthesis in ambiguous cases

A λ\lambdaλ-term λx.M\lambda x.Mλx.M is an anonymous function, also known as a λ\lambdaλ abstraction. It has 1 parameter xxx with a body MMM

Applications of the form MNM NMN are used to pass arguments to functions

we sometimes write λx1,…,xn.M\lambda x_1,\ldots,x_n.Mλx1​,…,xn​.M instead of λx1…λxn.M\lambda x_1 \ldots \lambda x_n.Mλx1​…λxn​.M

Officially a function with 1 parameter would take the form λx.M\lambda x.Mλx.M and a function with 2 parameters would take the form λx.λy.M\lambda x. \lambda y.Mλx.λy.M

We use brackets to remove ambiguity but in general, our precedence rule is that application has higher precedence over λ\lambdaλ
We also use an associativity rule, application is left associative, i.e: xyzxyzxyz stands for (xy)z(xy)z(xy)z

Abstract Syntax Trees

  • leaves are variables
  • nodes are either

Free and Bound Variables

Given a λ\lambdaλ- term, λx.M\lambda x.Mλx.M

  • xxx is the parameter of the function
  • the scope of xxx is the body MMM
  • the λ\lambdaλ-abstraction binds the "free" occurrences of xxx in MMM

A variable xxx is either

  • bound if it is i the scope of a λ\lambdaλ abstraction with a parameter xxx e.g λx.x\lambda x.xλx.x
  • free, otherwise e.g. λy.yx\lambda y.yxλy.yx

xxx can occur both free and bound in some cases: x(λx.x)x(\lambda x.x)x(λx.x)

We say that the occurrence of a bound variable xxx is bound to a λ\lambdaλ in a term MMM, fi the corresponding λ\lambdaλ-abstraction is the smallest λ\lambdaλ-abstraction in MMM with parameter xxx containing that bound occurrence

For example:
(λx.x)(λx.x)(\lambda x.x)(\lambda x.x)(λx.x)(λx.x)

  • xxx's 2nd2^{nd}2nd occurrence is bound to the 1st1^{st}1st λ\lambdaλ
  • xxx's 4th4^{th}4th occurrence is bound to the 2nd2^{nd}2nd λ\lambdaλ

The set of free variables occurring in a term is recursively computed as:

FV(x)={x}FV(λx.M)=FV(M) {x}FV(MN)=FV(M)∪FV(N)FV(x) = \{x\} \\ FV(\lambda x.M) = FV(M) \ \{x\} \\ FV(M N) = FV(M) \cup FV(N)FV(x)={x}FV(λx.M)=FV(M) {x}FV(MN)=FV(M)∪FV(N)

We can then say a variable xxx occurs free in MMM iff x∈FV(M)x \in FV(M)x∈FV(M)

A term MMM is closed if FV(M)=∅FV(M) = \emptyFV(M)=∅

α\alphaα-equivalence

Bound variable names are not important and can be renamed without changing the meaning of a term

Two terms MMM and NNN are α\alphaα-equivalent - denoted M≡αNM \equiv_{\alpha} NM≡α​N if they only differ by their bound variables, i.e

  • MMM and NNN have the same tree structure up to the leaves
  • MMM and NNN have the same occurrences of free variables
  • one superscript assignment can be used to label both MMM and NNN
