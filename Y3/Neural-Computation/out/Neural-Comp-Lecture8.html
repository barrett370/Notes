Impact of learning rate ε\varepsilonε on Stochastic Gradient Descent

  • Learning rate in SGD can have a marked impact on the optimisation time of a network

  • Often necessary to vary/ adjust the learning rate according to the specific model you are training.

  • If a learning rate that is too high is used then the model parameters vary wildly causing a large variance in the value of the loss function, J\mathcal{J}J

  • If too low a learning rate is used then the model stagnates far from the optimum conditions, it may converge given enough epochs but this would take exponentially long relative to the size of the learning rate

Conceptually, we can think of momentum in this context the same as in physics:

A Ball with position Θ\ThetaΘ and a velocity vvv influenced by two forces, one which pushes the ball in the opposite direction of the current gradient, and a viscous drag determined by a parameter α<1\alpha < 1α<1

The momentum smooths out the update steps. The size of the updates is proportional to how similar the previous gradients are.

Two Hyper-parameters

  • ε←\varepsilon \leftarrowε← learning rate
  • α←\alpha \leftarrowα← Factor which determines the influence of the past gradients on the current update of the parameter, typical values include 0.5, 0.9 or 0.99
